OES ID Extractor

Offline document-to-photo extraction pipeline

View on GitHub Get in Touch
PythonComputer VisionYOLOOCR Background RemovalPyInstallerOffline-first

The problem

OES Energy Services needed to produce staff ID cards from a stack of scanned PDFs and photo files — a manual process of opening each document, cropping the photo, cropping the signature, cleaning up the background, and saving files by hand for every single person. It was slow, repetitive, and easy to get wrong at scale.

Built during my internship

I designed and built this tool while interning at OES Energy Services, specifically to solve this internal workflow problem. It's now used in-house for ID card production.

What it does

  • Document parsing: automatically scans multi-page PDF documents or single images to locate photos, names, and signatures.
  • Auto-cropping & alignment: intelligently isolates personnel portraits and their matching signatures on detection.
  • Background removal: strips backgrounds from both portraits and raw signature scans — fully offline.
  • Image enhancement: sharpens and clarifies extracted photos to meet production-grade specs.
  • Automated named export: outputs two cleanly named files per person, matched for easy directory lookup.
  • Batch processing: drop multiple files into a queue and track progress through a native, responsive interface.

Why fully offline

Every part of the pipeline — detection, background removal, sharpening — runs locally with zero external API or cloud dependency. That wasn't a technical preference; it was a requirement. The documents involved contain personnel data, and OES needed a strict guarantee that nothing left the building. The one-time exception is the initial download of the background-removal model (u2net.onnx) on first launch, after which the app runs completely disconnected.

Shipping it

For non-technical end users, the tool ships as a single .exe — no Python install, no dependency wrangling. A build.bat script automates the whole developer-side lifecycle: spinning up an isolated virtual environment, syncing dependencies, resolving packaging conflicts, and bundling everything into a standalone binary with PyInstaller.

# first-time setup for developers
git clone https://github.com/onyedikachinzute/OES-ID-Extractor.git
cd OES-ID-Extractor
build.bat # sets up venv, installs deps, builds the .exe
