Panaview

Movie reservation & booking system

View on GitHub Get in Touch
PythonPostgreSQLTkinter psycopg2Admin Dashboards

The problem

Panaview is an SQL and Python-based movie reservation and booking application. It lets users browse available movies and showtimes, check ticket prices, pick seats with live availability, and walk away with a booking receipt — while giving admins the tools to manage the catalogue behind the scenes.

Screenshots

User-facing features

  • Browse movies and showtimes with visible ticket pricing.
  • Select and reserve seats with live seat availability, so two users can't double-book the same seat.
  • Generate booking receipts with the total cost calculated automatically.
  • Persisted bookings — every reservation is saved to the database, not just held in memory.

Admin-facing features

  • Add or delete movies and showtimes from a dedicated admin dashboard.
  • View seat occupancy per show at a glance.
  • Cascading cleanup: deleting a movie automatically removes its associated shows, so the database never ends up with orphaned records.

How it's structured

The codebase is split into three clear layers: db/ handles database initialization and schema edits, logic/ holds the functions that populate and query the database (adding movies, shows, and admin accounts), and ui/ contains the Tkinter interface — separate modules for the admin dashboard, user dashboard, login, registration, and the booking flow itself. All database access runs through psycopg2 against a PostgreSQL backend.

# running Panaview locally
git clone https://github.com/onyedikachinzute/movie-booking-app.git
cd movie-booking-app
pip install psycopg2
# set your PostgreSQL credentials in db/create_db1.py
python initdb.py
python main.py
